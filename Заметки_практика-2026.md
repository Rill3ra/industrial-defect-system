Задание: Применение искусственных нейронных сетей для решения прикладных задач классификации, детектирования, сегментации, видеоаналитики и трекинга в компьютерном зрении с обязательным сравнением нескольких архитектур.

1. Цель практики
Освоить полный цикл разработки прикладной системы компьютерного зрения: от постановки задачи и подготовки данных до обучения нескольких архитектур, экспериментального сравнения качества, анализа ошибок и внедрения лучшей модели в демонстрационное приложение с визуализацией результатов.
Этапы выполнения: 
1) Ознакомление с вариантом задания. Допускается выбор собственного варианта только по согласованию с руководителем практики.
2) Подбор и описание данных для выбранной прикладной задачи. Необходимо указать тип разметки, количество классов, объём выборки и ограничения использования данных.
3) Подготовка данных: разбиение на train/validation/test, проверка разметки, нормализация, аугментация и подготовка структуры проекта.
4) Выбор архитектур нейронных сетей. Для выбранной задачи необходимо обучить или дообучить минимум 5 различных архитектур и сравнить их между собой.
5) Проведение экспериментального сравнения. Для каждой архитектуры необходимо зафиксировать метрики качества, скорость инференса, размер модели и основные ошибки.
6) Интеграция лучшей модели в демонстрационный модуль. В программе должны отображаться исходные данные, результат работы модели, уверенность предсказания и краткая статистика.
7) Вывод статистики. Реализовать сохранение истории запусков или результатов в JSON/БД, а также генерацию краткого отчёта в PDF или Excel.
8) Подготовка итогового отчёта с анализом результатов, таблицей сравнения архитектур и примерами успешной и ошибочной работы модели.


2. Обязательные требования к сравнению архитектур
В каждом варианте студент обязан обучить или дообучить не менее пяти архитектур нейронных сетей. Простая замена размера одной и той же модели, например только YOLOv8n, YOLOv8s, YOLOv8m, не считается полноценным сравнением пяти архитектур. Необходимо сравнивать разные архитектурные подходы или разные семейства моделей, подходящие для выбранного типа задачи.

Для каждой архитектуры необходимо указать: схему обучения, размер входного изображения, число эпох, основные параметры обучения, метрики качества, среднее время обработки одного изображения или кадра, размер модели, а также краткий вывод о сильных и слабых сторонах модели.

3. Требования к оценке качества
1) Для классификации использовать accuracy, precision, recall, F1-score и confusion matrix.
2) Для детекции использовать mAP, precision, recall, количество ложных срабатываний и пропусков.
3) Для сегментации использовать IoU, Dice/F1, pixel accuracy и визуальный анализ масок.
4) Для трекинга использовать IDF1, MOTA/HOTA при наличии разметки, а также число потерь ID и визуальный анализ траекторий.
5) Для видеоаналитики дополнительно оценивать задержку обработки, стабильность результата по кадрам и качество работы на сложных сценах.

2. Где искать открытые датасеты

Kaggle Datasets	https://www.kaggle.com/datasets	Большой каталог пользовательских датасетов. Подходит для классификации, детекции, медицинских, спутниковых и бытовых CV-задач. Важно проверять лицензию, качество разметки и наличие train/test split.

Papers with Code Datasets	https://paperswithcode.com/datasets	Удобен для поиска датасетов по конкретной задаче и просмотра связанных статей, метрик и лидербордов.

3. Как понять, что датасет подходит
Перед началом обучения студент должен проверить датасет по нескольким критериям. Нельзя брать набор данных только потому, что он есть в открытом доступе.
Соответствие прикладной задаче: изображения должны быть похожи на те условия, где система будет применяться.
Наличие нужного типа разметки: для детекции нужны bounding boxes, для сегментации - маски, для трекинга - ID объектов во времени.
Достаточный объём данных: для учебного проекта можно использовать подвыборку, но она должна покрывать все классы.
Качество разметки: нужно открыть часть изображений и проверить, нет ли пропущенных объектов, неверных классов и смещённых рамок.
Разделение на train/validation/test: если оно отсутствует, студент должен выполнить его самостоятельно и описать принцип разбиения.
Баланс классов: если один класс встречается значительно чаще других, это нужно учитывать при обучении и анализе метрик.
Лицензия и ограничения: нужно проверить, разрешено ли использовать данные в учебных и исследовательских целях.
Отсутствие утечки данных: похожие или одинаковые изображения не должны одновременно попадать в train и test.
4. Какие архитектуры выбирать
Архитектуры нужно выбирать исходя из типа задачи. Нельзя сравнивать модели, которые решают разные задачи, например классификатор изображения и детектор объектов, если итоговая задача требует рамки объектов.
Тип задачи	Рекомендуемые современные архитектуры
Классификация изображений	ResNet, DenseNet, MobileNetV3, EfficientNet, Vision Transformer.
Детекция объектов	YOLO, Faster R-CNN, RetinaNet, SSD, RT-DETR/DETR.
Сегментация	U-Net, DeepLabV3+, SegFormer, Mask R-CNN, YOLO-seg.
Видео, действия и трекинг	3D ResNet, SlowFast, TimeSformer, VideoMAE, CNN+LSTM; для трекинга фиксируется одна модель детекции и сравниваются разные трекеры: ByteTrack, BoT-SORT, OC-SORT, StrongSORT, DeepOCSORT.
5. Отличия архитектур
5.1. Классификация изображений
Архитектура	Основная идея	Когда выбирать	На что обратить внимание
ResNet	Остаточные связи помогают обучать глубокие CNN без сильной деградации качества.	Как надёжный базовый вариант для большинства классификационных задач.	Хороший baseline, но может быть тяжелее мобильных моделей.
DenseNet	Каждый слой получает признаки от предыдущих слоёв, что усиливает переиспользование признаков.	Когда важно качество на небольших данных и детальных визуальных признаках.	Может требовать больше памяти.
MobileNetV3	Лёгкая CNN-архитектура для мобильных и быстрых приложений.	Когда важны скорость, небольшой размер модели и инференс на слабом устройстве.	Обычно уступает более крупным моделям по качеству.
EfficientNet	Сбалансированное масштабирование глубины, ширины и разрешения изображения.	Когда нужно хорошее соотношение качество/размер/скорость.	Важно правильно подобрать вариант B0-B7 или V2 под ресурсы.
Vision Transformer	Изображение разбивается на патчи, которые обрабатываются механизмом attention.	Когда есть предобученные веса и достаточно данных для дообучения.	На малых данных без transfer learning может проигрывать CNN.
5.2. Детекция объектов
Архитектура	Основная идея	Когда выбирать	На что обратить внимание
YOLO	Одностадийная детекция с высокой скоростью обработки.	Для real-time задач, видеоаналитики и демонстрационных приложений.	Разные версии YOLO не всегда считаются разными архитектурами; нужно обосновывать сравнение.
Faster R-CNN	Двухстадийный подход: сначала region proposals, затем классификация и уточнение рамок.	Когда важна точность, а скорость не является главным ограничением.	Сложнее и медленнее, чем большинство YOLO-моделей.
RetinaNet	Одностадийный детектор с focal loss для борьбы с дисбалансом фон/объект.	Когда много фона и есть проблема дисбаланса классов.	Может быть хорошим сравнительным baseline для детекции.
SSD	Быстрый одностадийный детектор с предсказанием на нескольких масштабах.	Для простых и быстрых детекционных проектов.	Часто хуже современных YOLO/RT-DETR на сложных сценах.
DETR/RT-DETR	Transformer-подход к детекции с объектными запросами; RT-DETR ориентирован на real-time.	Когда нужно сравнить CNN-based и transformer-based подходы.	Требует аккуратной настройки и оценки скорости.
5.3. Сегментация
Архитектура	Тип сегментации	Основная идея	Когда выбирать
U-Net	Семантическая/бинарная	Encoder-decoder со skip connections для точной локализации.	Медицина, трещины, дефекты, небольшие датасеты с масками.
DeepLabV3+	Семантическая	Atrous convolution и decoder для учёта контекста и границ.	Городские сцены, дороги, крупные области.
SegFormer	Семантическая	Transformer encoder и лёгкий MLP decoder.	Современное сравнение CNN и transformer-подходов.
Mask R-CNN	Instance	Детекция объектов и отдельная маска для каждого объекта.	Когда нужно различать отдельные экземпляры объектов.
YOLO-seg	Instance	Быстрая сегментация в стиле YOLO.	Когда важны скорость и демонстрация на видео.
5.4. Видеоаналитика и распознавание действий
Архитектура	Основная идея	Когда выбирать
3D ResNet	3D-свёртки обрабатывают пространственные и временные признаки одновременно.	Базовый вариант для классификации коротких видеоклипов.
SlowFast	Две ветви: медленная для семантики, быстрая для движения.	Когда действие зависит от движения и динамики.
TimeSformer	Transformer-attention по пространству и времени.	Для сравнения CNN и transformer-подходов на видео.
VideoMAE	Предобучение через восстановление замаскированных видеотокенов.	Когда данных немного и есть возможность использовать предобученную модель.
CNN+LSTM	CNN извлекает признаки кадров, LSTM анализирует последовательность.	Простой интерпретируемый baseline для учебного проекта.
5.5. Трекинг объектов
В задачах трекинга сравнение должно быть построено корректно: детектор фиксируется, а трекеры сравниваются на одинаковых детекциях или на одинаковой модели детекции. Иначе нельзя понять, улучшение связано с трекером или с детектором.
Трекер	Основная идея	Когда полезен
ByteTrack	Использует не только уверенные, но и низкоуверенные детекции для восстановления траекторий.	Хороший базовый современный трекер для людей и транспорта.
BoT-SORT	Улучшает ассоциацию объектов, может учитывать движение камеры и appearance-признаки.	Сцены с перекрытиями, движением камеры и плотным потоком.
OC-SORT	Улучшает motion-based tracking и ассоциацию на основе наблюдений.	Когда важна устойчивость движения без тяжёлого ReID.
StrongSORT	Использует сильные appearance/ReID-признаки для ассоциации объектов.	Когда объекты часто перекрываются или похожи по траектории.
DeepOCSORT	Сочетает идеи OC-SORT и appearance-признаков.	Для сравнения motion-only и appearance-assisted подходов.
6. Как честно сравнивать 5 архитектур
Сравнение архитектур должно быть воспроизводимым. Для всех моделей необходимо по возможности использовать одинаковые данные, одинаковое разбиение, одинаковые критерии оценки и фиксированное тестовое множество.
Использовать один и тот же train/validation/test split для всех моделей.
Фиксировать seed и параметры разбиения, чтобы результаты можно было повторить.
Описывать настройки обучения: эпохи, batch size, learning rate, optimizer, augmentations, input size.
Для transfer learning явно указать, какие веса использовались и какие слои дообучались.
Сравнивать не только качество, но и скорость инференса, размер модели и удобство внедрения.
В трекинге фиксировать детектор и сравнивать именно трекеры.
Не подбирать test set под лучшую модель; финальная оценка должна проводиться один раз после выбора модели.
Показывать минимум три успешных и три ошибочных примера работы лучшей модели.

Не указаны параметры обучения, поэтому эксперимент нельзя повторить.
Скорость модели измеряется без указания устройства: CPU/GPU, видеокарта, размер входа.

Создаем файл

src/data/split_data.py

from pathlib import Path
from sklearn.model_selection import train_test_split
import json
import random

SEED = 42

random.seed(SEED)

ROOT = Path("data/raw/mvtec_ad")

CATEGORIES = [
    "screw",
    "metal_nut"
]

samples = []

for category in CATEGORIES:

    # good train
    train_good = ROOT / category / "train" / "good"

    for img in train_good.glob("*.*"):
        samples.append(
            {
                "path": str(img),
                "label": 0,
                "category": category
            }
        )

    # defects from test
    test_dir = ROOT / category / "test"

    for defect_folder in test_dir.iterdir():

        if defect_folder.name == "good":
            continue

        for img in defect_folder.glob("*.*"):
            samples.append(
                {
                    "path": str(img),
                    "label": 1,
                    "category": category
                }
            )

print(f"Total samples: {len(samples)}")

src/data/dataset.py
import json

from PIL import Image

from torch.utils.data import Dataset


class DefectDataset(Dataset):

    def __init__(
        self,
        json_path,
        transform=None
    ):

        with open(json_path, "r") as f:
            self.samples = json.load(f)

        self.transform = transform

    def __len__(self):
        return len(self.samples)

    def __getitem__(self, idx):

        sample = self.samples[idx]

        image = Image.open(
            sample["path"]
        ).convert("RGB")

        label = sample["label"]

        if self.transform:
            image = self.transform(image)

        return {
            "image": image,
            "label": label,
            "category": sample["category"],
            "defect_type": sample["defect_type"]
        }

import torch
import torch.nn as nn
import torch.optim as optim

from torch.utils.data import DataLoader
from sklearn.metrics import accuracy_score, f1_score, classification_report

from src.data.dataset import DefectDataset
from src.data.transforms import get_train_transforms, get_val_transforms

from torchvision import models

import os


# =========================
# DEVICE (Mac MPS support)
# =========================
device = torch.device("mps" if torch.backends.mps.is_available() else "cpu")
print("Device:", device)


# =========================
# DATA
# =========================
train_dataset = DefectDataset(
    json_path="data/splits/train.json",
    transform=get_train_transforms()
)

val_dataset = DefectDataset(
    json_path="data/splits/val.json",
    transform=get_val_transforms()
)

train_loader = DataLoader(train_dataset, batch_size=16, shuffle=True)
val_loader = DataLoader(val_dataset, batch_size=16, shuffle=False)


# =========================
# MODEL (ResNet50)
# =========================
model = models.resnet50(weights=models.ResNet50_Weights.DEFAULT)
model.fc = nn.Linear(model.fc.in_features, 2)  # binary classification
model = model.to(device)


# =========================
# LOSS + OPTIMIZER
# =========================
criterion = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=1e-4)


# =========================
# TRAIN FUNCTION
# =========================
def train_one_epoch():
    model.train()

    all_preds = []
    all_labels = []

    total_loss = 0

    for batch in train_loader:
        images = batch["image"].to(device)
        labels = batch["label"].to(device)

        optimizer.zero_grad()

        outputs = model(images)
        loss = criterion(outputs, labels)

        loss.backward()
        optimizer.step()

        total_loss += loss.item()

        preds = torch.argmax(outputs, dim=1)

        all_preds.extend(preds.cpu().numpy())
        all_labels.extend(labels.cpu().numpy())

    acc = accuracy_score(all_labels, all_preds)
    f1 = f1_score(all_labels, all_preds)

    return total_loss / len(train_loader), acc, f1


# =========================
# VALIDATION
# =========================
def evaluate():
    model.eval()

    all_preds = []
    all_labels = []

    with torch.no_grad():
        for batch in val_loader:
            images = batch["image"].to(device)
            labels = batch["label"].to(device)

            outputs = model(images)
            preds = torch.argmax(outputs, dim=1)

            all_preds.extend(preds.cpu().numpy())
            all_labels.extend(labels.cpu().numpy())

    acc = accuracy_score(all_labels, all_preds)
    f1 = f1_score(all_labels, all_preds)

    return acc, f1, all_labels, all_preds


# =========================
# TRAIN LOOP
# =========================
EPOCHS = 10
best_f1 = 0

os.makedirs("runs/checkpoints", exist_ok=True)

for epoch in range(EPOCHS):

    train_loss, train_acc, train_f1 = train_one_epoch()
    val_acc, val_f1, y_true, y_pred = evaluate()

    print(f"\nEpoch {epoch+1}/{EPOCHS}")
    print(f"Train Loss: {train_loss:.4f}")
    print(f"Train Acc: {train_acc:.4f} | F1: {train_f1:.4f}")
    print(f"Val Acc: {val_acc:.4f} | Val F1: {val_f1:.4f}")

    # save best model
    if val_f1 > best_f1:
        best_f1 = val_f1
        torch.save(model.state_dict(), "runs/checkpoints/resnet50.pth")
        print("Saved best model")


# =========================
# FINAL REPORT
# =========================
print("\nClassification report:")
print(classification_report(y_true, y_pred))

```python
import torch

from torchvision import models

from src.data.transforms import get_train_transforms, get_val_transforms


# =========================
# DEVICE
# =========================
device = torch.device("mps" if torch.backends.mps.is_available() else "cpu")
print("Device:", device)


# =========================
# DATA
# =========================
train_dataset = DefectDataset(
    json_path="data/splits/train.json",
    transform=get_train_transforms()
)

    json_path="data/splits/val.json",
    transform=get_val_transforms()
)

train_loader = DataLoader(train_dataset, batch_size=16, shuffle=True)
val_loader = DataLoader(val_dataset, batch_size=16, shuffle=False)


# =========================
# CLASS WEIGHTS (IMPORTANT)
# =========================
labels = [item["label"] for item in train_dataset.samples]

# TRAIN
# =========================

    all_preds = []
    all_labels = []

    total_loss = 0

    for batch in train_loader:
        images = batch["image"].to(device)
        labels = batch["label"].to(device)

        optimizer.zero_grad()

        outputs = model(images)
        loss = criterion(outputs, labels)

        loss.backward()
        optimizer.step()

        total_loss += loss.item()

        preds = torch.argmax(outputs, dim=1)

        all_preds.extend(preds.cpu().numpy())
        all_labels.extend(labels.cpu().numpy())

    acc = accuracy_score(all_labels, all_preds)
    f1 = f1_score(all_labels, all_preds)

    return total_loss / len(train_loader), acc, f1

# EVAL
# =========================

    model.eval()

    all_preds = []
    all_labels = []

    with torch.no_grad():
        for batch in val_loader:
            images = batch["image"].to(device)
            labels = batch["label"].to(device)

            outputs = model(images)
            preds = torch.argmax(outputs, dim=1)

            all_preds.extend(preds.cpu().numpy())
            all_labels.extend(labels.cpu().numpy())

    acc = accuracy_score(all_labels, all_preds)
    f1 = f1_score(all_labels, all_preds)

    return acc, f1, all_labels, all_preds
```

# 🧠 2. Код PatchCore (упрощённый, рабочий)

```python
import torch
import torch.nn as nn
import torchvision.models as models
import torch.nn.functional as F
import numpy as np


class PatchCore(nn.Module):
    def __init__(self, device):
        super().__init__()

        self.device = device

        # backbone (feature extractor)
        self.backbone = models.resnet18(weights=None)
        self.backbone = nn.Sequential(*list(self.backbone.children())[:-2])

        self.backbone.to(device)
        self.backbone.eval()

        self.memory_bank = None

    def extract_features(self, x):
        with torch.no_grad():
            feats = self.backbone(x)
            feats = feats.mean(dim=[2, 3])  # global pooling
        return feats

    def build_memory_bank(self, dataloader):
        features = []

        for batch in dataloader:
            x = batch["image"].to(self.device)
            f = self.extract_features(x)
            features.append(f.cpu())

        self.memory_bank = torch.cat(features, dim=0)

        print("Memory bank size:", self.memory_bank.shape)

    def compute_anomaly_score(self, x):
        f = self.extract_features(x)

        dists = torch.cdist(f.cpu(), self.memory_bank)
        score = torch.min(dists, dim=1)[0]

        return score
```

```python
)
```

    train_labels = []

    total_loss = 0

    for batch in train_loader:

        optimizer.zero_grad()

        outputs = model(images)

        loss = criterion(

        total_loss += loss.item()

        torch.save(

from torchvision import models
import torch.nn as nn


def get_model(model_name: str):

    if model_name == "resnet50":

        model = models.resnet50(weights=None)
        model.fc = nn.Linear(
            model.fc.in_features,
            2
        )

    elif model_name == "densenet121":

        model = models.densenet121(weights=None)
        model.classifier = nn.Linear(
            model.classifier.in_features,
            2
        )

    elif model_name == "efficientnet_b0":

        model = models.efficientnet_b0(weights=None)
        model.classifier[1] = nn.Linear(
            model.classifier[1].in_features,
            2
        )

    elif model_name == "vit_b16":

        model = models.vit_b_16(weights=None)
        model.heads.head = nn.Linear(
            model.heads.head.in_features,
            2
        )

    else:
        raise ValueError(
            f"Unknown model: {model_name}"
        )

    return model

import argparse

import torch

from torch.utils.data import DataLoader

from sklearn.metrics import (
    classification_report,
    confusion_matrix,
    accuracy_score,
    precision_score,
    recall_score,
    f1_score
)

from src.data.dataset import DefectDataset
from src.data.transforms import get_val_transforms
from src.models.model_factory import get_model


parser = argparse.ArgumentParser()

parser.add_argument(
    "--model",
    type=str,
    required=True
)

args = parser.parse_args()

MODEL_NAME = args.model


device = torch.device(
    "mps"
    if torch.backends.mps.is_available()
    else "cpu"
)

print("Device:", device)
print("Model:", MODEL_NAME)


dataset = DefectDataset(
    "data/splits/test.json",
    transform=get_val_transforms()
)

loader = DataLoader(
    dataset,
    batch_size=16,
    shuffle=False
)


model = get_model(MODEL_NAME)

model.load_state_dict(
    torch.load(
        f"runs/checkpoints/{MODEL_NAME}.pth",
        map_location=device
    )
)

model.to(device)
model.eval()


preds_all = []
labels_all = []


with torch.no_grad():

    for batch in loader:

        images = batch["image"].to(device)
        labels = batch["label"].to(device)

        outputs = model(images)

        preds = outputs.argmax(1)

        preds_all.extend(
            preds.cpu().numpy()
        )

        labels_all.extend(
            labels.cpu().numpy()
        )


print("\nAccuracy:",
      accuracy_score(labels_all, preds_all))

print("Precision:",
      precision_score(labels_all, preds_all))

print("Recall:",
      recall_score(labels_all, preds_all))

print("F1:",
      f1_score(labels_all, preds_all))

print("\nClassification Report\n")

print(
    classification_report(
        labels_all,
        preds_all
    )
)

print("\nConfusion Matrix\n")

print(
    confusion_matrix(
        labels_all,
        preds_all
    )
)

# ✅ 3. ЕДИНЫЙ pipeline (главный фикс)

## 📌 `src/inference/pipeline.py` (ОКОНЧАТЕЛЬНАЯ версия)

```python
import torch
import numpy as np
from PIL import Image

from src.models.patchcore_extractor import PatchCoreExtractor
from src.data.transforms import get_val_transforms
from src.utils.heatmap import patchcore_heatmap
from src.models.patchcore_postprocess import heatmap_to_bbox
from src.models.model_factory import get_model


device = torch.device("mps" if torch.backends.mps.is_available() else "cpu")

transform = get_val_transforms()

# ---------------- models ----------------
classifier = get_model("densenet121").to(device)
classifier.load_state_dict(torch.load(
    "runs/checkpoints/densenet121.pth",
    map_location=device
))
classifier.eval()

extractor = PatchCoreExtractor().to(device)
extractor.eval()

patchcore_bank = torch.load(
    "runs/checkpoints/patchcore_memory_bank.pt",
    map_location=device
)

patchcore_threshold = 0.08


# ---------------- core functions ----------------
def extract_feature_map(x):
    with torch.no_grad():
        return extractor(x)


def patchcore_score(feat_map, bank):
    B, C, H, W = feat_map.shape
    patches = feat_map.permute(0, 2, 3, 1).reshape(-1, C)
    dist = torch.cdist(patches, bank)
    return dist.min().item()


# ---------------- main ----------------
def predict(image_path):

    image = Image.open(image_path).convert("RGB")
    x = transform(image).unsqueeze(0).to(device)

    with torch.no_grad():
        logits = classifier(x)
        probs = torch.softmax(logits, dim=1)[0]
        classifier_prob = probs[1].item()

    feat_map = extract_feature_map(x)

    score = patchcore_score(feat_map, patchcore_bank)
    heatmap = patchcore_heatmap(feat_map, patchcore_bank)

    bbox = heatmap_to_bbox(heatmap, threshold_percentile=95)

    # fusion logic
    if score > patchcore_threshold:
        final_label = 1
        decision = "defect"
    else:
        final_label = int(classifier_prob > 0.5)
        decision = "defect" if final_label == 1 else "normal"

    return {
        "final_label": final_label,
        "decision": decision,
        "classifier_prob": float(classifier_prob),
        "patchcore_score": float(score),
        "heatmap": heatmap,
        "bbox": bbox
    }
```

# ✅ 4. FIX utils (ВАЖНО)

## 📌 `src/utils/heatmap.py`

Убедись что там есть:

```python
import torch


def patchcore_heatmap(feat_map, bank):

    B, C, H, W = feat_map.shape
    patches = feat_map.permute(0, 2, 3, 1).reshape(-1, C)

    dist = torch.cdist(patches, bank)
    min_dist = dist.min(dim=1)[0]

    heatmap = min_dist.reshape(H, W)

    heatmap = (heatmap - heatmap.min()) / (heatmap.max() - heatmap.min() + 1e-8)

    return heatmap
```

```python
import torch
import numpy as np
from PIL import Image

from src.models.patchcore_extractor import PatchCoreExtractor
from src.data.transforms import get_val_transforms
from src.utils.heatmap import patchcore_heatmap
from src.models.patchcore_postprocess import heatmap_to_bbox
from src.models.model_factory import get_model


device = torch.device("mps" if torch.backends.mps.is_available() else "cpu")
transform = get_val_transforms()

# ---------------- models ----------------
classifier = get_model("densenet121").to(device)
classifier.load_state_dict(torch.load(
    "runs/checkpoints/densenet121.pth",
    map_location=device
))
classifier.eval()

extractor = PatchCoreExtractor().to(device)
extractor.eval()

patchcore_bank = torch.load(
    "runs/checkpoints/patchcore_memory_bank.pt",
    map_location=device
)

patchcore_threshold = 0.08


# ---------------- core ----------------
def extract_feature_map(x):
    with torch.no_grad():
        return extractor(x)


def patchcore_score(feat_map, bank):
    B, C, H, W = feat_map.shape
    patches = feat_map.permute(0, 2, 3, 1).reshape(-1, C)
    dist = torch.cdist(patches, bank)
    return dist.min().item()


# ---------------- MAIN (FIXED) ----------------
def predict(image: Image.Image):

    image = image.convert("RGB")
    x = transform(image).unsqueeze(0).to(device)

    with torch.no_grad():
        logits = classifier(x)
        probs = torch.softmax(logits, dim=1)[0]
        classifier_prob = probs[1].item()

    feat_map = extract_feature_map(x)

    score = patchcore_score(feat_map, patchcore_bank)
    heatmap = patchcore_heatmap(feat_map, patchcore_bank)

    bbox = heatmap_to_bbox(heatmap, threshold_percentile=95)

    if score > patchcore_threshold:
        final_label = 1
        decision = "defect"
    else:
        final_label = int(classifier_prob > 0.5)
        decision = "defect" if final_label == 1 else "normal"

    return {
        "final_label": final_label,
        "decision": decision,
        "classifier_prob": float(classifier_prob),
        "patchcore_score": float(score),
        "heatmap": heatmap,
        "bbox": bbox
    }
```

```python
import torch
import numpy as np
from scipy.ndimage import gaussian_filter
from skimage.measure import label, regionprops


def heatmap_to_bbox(heatmap: torch.Tensor, threshold_percentile: float = 95):

    hm = heatmap.detach().cpu().numpy()

    hm = gaussian_filter(hm, sigma=2)

    thr = np.percentile(hm, threshold_percentile)
    binary = hm > thr

    labeled = label(binary)

    if labeled.max() == 0:
        return None

    regions = regionprops(labeled)
    largest = max(regions, key=lambda r: r.area)

    minr, minc, maxr, maxc = largest.bbox

    return {
        "x1": int(minc),
        "y1": int(minr),
        "x2": int(maxc),
        "y2": int(maxr),
        "score": float(hm.max())
    }
```
